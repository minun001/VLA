단안 dashcam 프레임이 차량 탐지, ego distance와 bbox gap edge, pseudo-3D 맥락, TGCA 기반 blue prediction cue와 horizon prior로 변환되는 과정을 보여줍니다.

Monocular input Vehicle detection Gap edges Pseudo-3D Blue cue prior
시각화 수준의 결과 · ego distance/gap은 pseudo-metric 추정 · blue cue는 control command, driving decision, ground-truth future가 아닌 prior

Pipeline Overview

이 페이지가 보여주는 것

Input 단안 dashcam 입력 프레임
Graph ego distance / confirmed gap / projected fallback edge 구성
Scene 추정 거리와 pseudo-3D 장면 표현
Output TGCA 기반 blue prediction cue와 1-5s horizon prior
01

원본 이미지

perception 또는 graph 구성이 적용되기 전의 단안 dashcam 입력 장면입니다.

02

탐지된 객체 bbox

전방 차량 후보를 bounding box로 표시합니다.

03

bbox + gap edges + pseudo-3D 장면

ego distance, confirmed edge gap, projected fallback 등 bbox 기반 거리 단서를 2D와 pseudo-3D 뷰에 함께 표시합니다.

  • 초록색: ego distance
  • 하늘색: confirmed edge gap
  • 노란 점선: projected fallback
  • 빨간색: negative signed gap
  • 오른쪽 패널: pseudo-3D 맥락
04

TGCA floating ego prediction cues

TGCA 출력의 raw prediction을 파란 floating direction cue로 표시합니다. 2D 뷰에서는 cue들을 연결한 궤적을 그리지 않고, 오른쪽 pseudo-3D 패널에는 raw TGCA prediction line을 함께 보여줍니다.

  • 파란색: raw TGCA prediction / floating direction cue
  • 상단 HUD: without connecting trajectory, not AutoSteer corrected
05

1초부터 5초까지 TGCA horizon cues

future ego horizon prior를 파란 elevated cue로 표시하고, 오른쪽 pseudo-3D 패널에는 1초부터 5초까지의 horizon label을 함께 보여줍니다.

  • 파란색: horizon label과 elevated direction cue

Takeaway

전방 객체 맥락에서 해석 가능한 ego prediction prior로

이 페이지는 시각화 수준의 연구 결과를 보여줍니다. 탐지된 차량 맥락을 ego distance와 bbox gap relation으로 구성하고, pseudo-3D 맥락으로 투영한 뒤, TGCA 기반 blue prediction/horizon cue를 표시합니다. 이 결과는 validated future label, planner output, autonomous-driving control signal이 아니라 해석 가능한 prediction prior로 읽어야 합니다.
