원본 객체 탐지 Lane Edge + 3D TGCA Overlay

전방 객체 맥락에서 TGCA ego trajectory prior까지

차량 탐지, lane context, relation graph, TGCA horizon prior를 단계별로 보여줍니다.

Monocular input Vehicle detection Lane context Gap edges Pseudo-3D TGCA prior
거리와 궤적은 시각화 prior이며, 제어 명령이나 정답 label이 아닙니다.

Pipeline Overview

이 페이지가 보여주는 것

Input 단안 dashcam 입력 프레임
Graph ego distance / confirmed edge gap / fallback distance
Lane ego lane 차량 여부와 LC 후보 판단
Output TGCA horizon prior와 overlay view

Visualization Legend

색상 의미 요약

Relation graph

  • Green solid: ego distance
  • Cyan solid: confirmed edge gap
  • Amber dashed: projected fallback distance
  • Red: negative signed gap

Trajectory

  • Blue marker: Raw TGCA prediction
  • 1s-5s: prediction horizon

Caution

All distances and trajectories are priors, not GT or control.

01

원본 이미지

bbox, edge, trajectory가 없는 입력 frame입니다.

02

객체 bbox

초록 bbox는 lead vehicle, 흰 bbox는 일반 vehicle입니다.

03

ego lane

ego-lane 차량/LC 후보를 temporal KG에 보냅니다.

  • 청록 실선: ego lane boundary
  • 청록 mask: ego lane area
  • KG cue: in_ego_lane / LC_into_ego_lane
04

edge + pseudo-3D

relation edge를 2D와 pseudo-3D로 봅니다.

  • 초록: ego distance
  • 청록: confirmed edge gap
  • 노랑 점선: fallback distance
  • 빨강: negative gap
  • 오른쪽: pseudo-3D context
05

TGCA 기본 horizon

05 이미지는 1s-5s Raw TGCA horizon prior입니다.

  • 파랑: Raw TGCA horizon
  • 1s-5s: prediction step
  • 정답/제어가 아닌 prior
06

TGCA overlay view

05-1 이미지는 같은 horizon을 overlay로 강조합니다.

  • 파랑: Raw TGCA horizon
  • Overlay: marker 가독성 강조
  • validated future label이 아닌 prior

Takeaway

전방 객체 맥락에서 해석 가능한 ego trajectory prior로

bbox, lane context, relation edge, TGCA horizon을 temporal ontology KG cue로 연결합니다. 결과는 prior이며 정답/제어가 아닙니다.
